sqlmap简介
{
当给sqlmap这么一个url的时候，它会：
{
	1、判断可注入的参数
	2、判断可以用那种SQL注入技术来注入
	3、识别出哪种数据库
	4、根据用户选择，读取哪些数据
}
sqlmap支持五种不同的注入模式：
{
1、基于布尔的盲注，即可以根据返回页面判断条件真假的注入。
2、基于时间的盲注，即不能根据页面返回内容判断任何信息，用条件语句查看时间延迟语句是否执行（即页面返回时间是否增加）来判断。
3、基于报错注入，即页面会返回错误信息，或者把注入的语句的结果直接返回在页面中。
4、联合查询注入，可以使用union的情况下的注入。
5、堆查询注入，可以同时执行多条语句的执行时的注入。
}

sqlmap性能
{
sqlmap支持的数据库有：MySQL, Oracle, PostgreSQL, Microsoft SQL Server, Microsoft Access, IBM DB2, SQLite, Firebird, Sybase和SAP MaxDB

可以提供一个简单的URL，Burp或WebScarab请求日志文件，文本文档中的完整http请求或者Google的搜索，匹配出结果页面，也可以自己定义一个正则来判断那个地址去测试。

测试GET参数，POST参数，HTTP Cookie参数，HTTP User-Agent头和HTTP Referer头来确认是否有SQL注入，它也可以指定用逗号分隔的列表的具体参数来测试。

可以设定HTTP(S)请求的并发数，来提高盲注时的效率。
}

sqlmap详细信息
{
	如果你想观察sqlmap对一个点是进行了怎样的尝试判断以及读取数据的，可以使用-v参数。共有七个等级，默认为1：
	{
		0、只显示python错误以及严重的信息。
		1、同时显示基本信息和警告信息。（默认）
		2、同时显示debug信息。
		3、同时显示注入的payload。
		4、同时显示HTTP请求。
		5、同时显示HTTP响应头。
		6、同时显示HTTP响应页面。
		如果你想看到sqlmap发送的测试payload最好的等级就是3。
	}

***********高级用法*************
parameter参数：
	-p name //多个参数如index.php?n_id=1&name=2&data=2020 我们想指定name参数进行注入
	-g "google语法" Cdump-all Cbatch #google搜索注入点自动 跑出 所有字段,需保证google.com能正常访问
	C-technique   测试指定注入类型\使用的技术,不加参数默认测试所有注入技术
		?B: 基于布尔的SQL 盲注
		?E: 基于显错sql 注入
		?U: 基于UNION 注入
		?S: 叠层sql 注入
		?T: 基于时间盲注
	C-tamper 通过编码绕过WEB 防火墙（WAF）Sqlmap 默认用char()
		Ctamper 插件所在目录:\sqlmap-dev\tamper

Options（选项）：
    Cversion 显示程序的版本号并退出
    -h, Chelp 显示此帮助消息并退出
    -v VERBOSE 详细级别：0-6（默认为1）

Target（目标）：
	以下至少需要设置其中一个选项，设置目标URL。
		-d DIRECT 直接连接到数据库。
		-u URL, Curl=URL 目标URL。
		-l LIST 从Burp或WebScarab代理的日志中解析目标。
		-r REQUESTFILE 从一个文件中载入HTTP请求。
		-g GOOGLEDORK 处理Google dork的结果作为目标URL。
		-c CONFIGFILE 从INI配置文件中加载选项。

Request（请求）：
	这些选项可以用来指定如何连接到目标URL。
		-Cdata=DATA 通过POST发送的数据字符串
		-Ccookie=COOKIE HTTP Cookie头
		C-cookie-urlencode URL 编码生成的cookie注入
		Cdrop-set-cookie 忽略响应的Set C Cookie头信息
		Cuser-agent=AGENT 指定 HTTP User C Agent头
		Crandom-agent 使用随机选定的HTTP User C Agent头
		Creferer=REFERER 指定 HTTP Referer头
		Cheaders=HEADERS 换行分开，加入其他的HTTP头
		Cauth-type=ATYPE HTTP身份验证类型（基本，摘要或NTLM）(Basic, Digest or NTLM)
		Cauth-cred=ACRED HTTP身份验证凭据（用户名:密码）
		Cauth-cert=ACERT HTTP认证证书（key_file，cert_file）
		C-proxy=PROXY 使用HTTP代理连接到目标URL
		Cproxy-cred=PCRED HTTP代理身份验证凭据（用户名：密码）
		Cignore-proxy 忽略系统默认的HTTP代理
		Cdelay=DELAY 在每个HTTP请求之间的延迟时间，单位为秒
		Ctimeout=TIMEOUT 等待连接超时的时间（默认为30秒）
		Cretries=RETRIES 连接超时后重新连接的时间（默认3）
		Cscope=SCOPE 从所提供的代理日志中过滤器目标的正则表达式
		Csafe-url=SAFURL 在测试过程中经常访问的url地址
		Csafe-freq=SAFREQ 两次访问之间测试请求，给出安全的URL

Enumeration（枚举）：
这些选项可以用来列举后端数据库管理系统的信息、表中的结构和数据。此外，您还可以运行您自己的SQL语句。
    -b, Cbanner 检索数据库管理系统的标识
    Ccurrent-user 检索数据库管理系统当前用户
    Ccurrent-db 检索数据库管理系统当前数据库
    Cis-dba 检测DBMS当前用户是否DBA
    C-users 枚举数据库管理系统用户
    C-passwords 枚举数据库管理系统用户密码哈希
    C-privileges 枚举数据库管理系统用户的权限
    C-roles 枚举数据库管理系统用户的角色
    C-dbs 枚举数据库管理系统数据库
    -D DBname //要进行枚举的指定数据库名
    -T TBLname 要进行枚举的指定数据库表（如：-T tablename Ccolumns）
    C-tables 枚举的DBMS数据库中的表
    C-columns 枚举DBMS数据库表列
    C-dump 转储数据库管理系统的数据库中的表项
    C-dump-all 转储所有的DBMS数据库表中的条目
    Csearch 搜索列（S），表（S）和/或数据库名称（S）
    -C COL 要进行枚举的数据库列
    -U USER 用来进行枚举的数据库用户
    Cexclude-sysdbs 枚举表时排除系统数据库
    Cstart=LIMITSTART 第一个查询输出进入检索
    Cstop=LIMITSTOP 最后查询的输出进入检索
    Cfirst=FIRSTCHAR 第一个查询输出字的字符检索
    Clast=LASTCHAR 最后查询的输出字字符检索
    Csql-query=QUERY 要执行的SQL语句
    Csql-shell 提示交互式SQL的shell

Optimization（优化）：
这些选项可用于优化SqlMap的性能。
    -o 开启所有优化开关
    Cpredict-output 预测常见的查询输出
    Ckeep-alive 使用持久的HTTP（S）连接
    Cnull-connection 从没有实际的HTTP响应体中检索页面长度
    Cthreads=THREADS 最大的HTTP（S）请求并发量（默认为1）

Injection（注入）：
这些选项可以用来指定测试哪些参数， 提供自定义的注入payloads和可选篡改脚本。
    -p TESTPARAMETER 可测试的参数（S）
    C-dbms=DBMS 强制后端的DBMS为此值
    C-os=OS 强制后端的DBMS操作系统为这个值
    C-prefix=PREFIX 注入payload字符串前缀
    C-suffix=SUFFIX 注入payload字符串后缀
    C-tamper=TAMPER 使用给定的脚本（S）篡改注入数据(对payload进行字符替换)

Detection（检测）：
这些选项可以用来指定在SQL盲注时如何解析和比较HTTP响应页面的内容。
    C-level=LEVEL 执行测试的等级（1-5，默认为1）
    C-risk=RISK 执行测试的风险（0-3，默认为1）
    C-string=STRING 查询时有效时在页面匹配字符串
    C-regexp=REGEXP 查询时有效时在页面匹配正则表达式
    C-text-only 仅基于在文本内容比较网页

Techniques（技巧）：
这些选项可用于调整具体的SQL注入测试。
    C-technique=TECH SQL注入技术测试（默认BEUST）
    C-time-sec=TIMESEC DBMS响应的延迟时间（默认为5秒）
    C-union-cols=UCOLS 定列范围用于测试UNION查询注入
    C-union-char=UCHAR 用于暴力猜解列数的字符

Fingerprint（指纹）：
    -f, Cfingerprint 执行检查广泛的DBMS版本指纹

Brute force（蛮力）：
这些选项可以被用来运行蛮力检查。
    Ccommon-tables 检查存在共同表
    Ccommon-columns 检查存在共同列


User-defined function injection(用户自定义函数注入)：
这些选项可以用来创建用户自定义函数。
	Cudf-inject 注入用户自定义函数
	Cshared-lib=SHLIB 共享库的本地路径
	
File system access（访问文件系统）：
这些选项可以被用来访问后端数据库管理系统的底层文件系统。
    Cfile-read=RFILE 从后端的数据库管理系统文件系统读取文件
    Cfile-write=WFILE 编辑后端的数据库管理系统文件系统上的本地文件
    Cfile-dest=DFILE 后端的数据库管理系统写入文件的绝对路径

Operating system access（操作系统访问）：
这些选项可以用于访问后端数据库管理系统的底层操作系统。
    Cos-cmd=OSCMD 执行操作系统命令
    Cos-shell 交互式的操作系统的shell
    Cos-pwn 获取一个OOB shell，meterpreter或VNC
    Cos-smbrelay 一键获取一个OOB shell，meterpreter或VNC
    Cos-bof 存储过程缓冲区溢出利用
    Cpriv-esc 数据库进程用户权限提升
    Cmsf-path=MSFPATH Metasploit Framework本地的安装路径
    Ctmp-path=TMPPATH 远程临时文件目录的绝对路径

Windows注册表访问：
这些选项可以被用来访问后端数据库管理系统Windows注册表。
    Creg-read 读一个Windows注册表项值
    Creg-add 写一个Windows注册表项值数据
    Creg-del 删除Windows注册表键值
    Creg-key=REGKEY Windows注册表键
    Creg-value=REGVAL Windows注册表项值
    Creg-data=REGDATA Windows注册表键值数据
    Creg-type=REGTYPE Windows注册表项值类型

一般的工作参数：
这些选项可以用来设置一些一般的工作参数。
    -t TRAFFICFILE 记录所有HTTP流量到一个文本文件中
    -s SESSIONFILE 保存和恢复检索会话文件的所有数据
    Cflush-session 刷新当前目标的会话文件
    Cfresh-queries 忽略在会话文件中存储的查询结果
    Ceta 显示每个输出的预计到达时间
    Cupdate 更新SqlMap
    Csave file保存选项到INI配置文件
    Cbatch 从不询问用户输入，使用所有默认配置。

Miscellaneous（杂项）：
    Cbeep 发现SQL注入时提醒
    Ccheck-payload IDS对注入payloads的检测测试
    Ccleanup SqlMap具体的UDF和表清理DBMS
    Cforms 对目标URL的解析和测试形式
    Cgpage=GOOGLEPAGE 从指定的页码使用谷歌dork结果
    Cpage-rank Google dork结果显示网页排名（PR）
    Cparse-errors 从响应页面解析数据库管理系统的错误消息
    Creplicate 复制转储的数据到一个sqlite3数据库
    Ctor 使用默认的Tor（Vidalia/ Privoxy/ Polipo）代理地址
    Cwizard 给初级用户的简单向导界面
